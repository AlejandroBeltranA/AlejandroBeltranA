When to Stop

Operational reliability and enforceable accountability for agentic AI in government deployment.

Alejandro Beltran · Local Threshold Evaluation (LTE)

Agentic AI systems can become operationally unreliable before they produce a single visible hallucination. LTE is a pre-deployment harness that catches the drift early — and records what a deploying organisation had reason to know before it went live.

Three real harness runs, one protocol. Mistral-7B returns a valid five-bullet summary on every turn, yet its per-step latency climbs past the ceiling and the harness aborts. SmolLM never produces the format and collapses into repetition within three turns. Claude Haiku holds all thirty. The failure here is invisible in the text.

Why it matters

Pre-deployment governance stops at procurement. Risk classifications, model cards, and demonstration cases judge a system under assessment conditions. Deployment is a different setting: context accumulates across a long, multi-turn session, and the model's behaviour shifts as the window fills.

That gap is where the dangerous failures occur. When harm surfaces, liability turns on one question: what did the deploying organisation have reason to know about the model's reliability, and when? No standard instrument produces that record, so responsibility defaults downward — onto the analyst running a pipeline they neither designed nor controlled.

LTE produces the record. A pre-deployment run that returns an explicit signal, and is retained, turns a deployment into a traceable decision. The aim is narrow: make the evidence exist, so the choice to go live leaves a mark.

What LTE watches

Five operational failure modes. Each is visible to instrumentation well before any output would read as wrong to a human reviewer.

Over-expansion
Produces more than the task asks for. Bloats latency, burns the token budget, breaks downstream parsers.
Near-cap pressure
Pushes against the output limit; structure breaks at the tail — a JSON object cut off before its closing brace.
Context decay
Mishandles carried state as the window fills. The text stays plausible; the determinations drift.
Latency cliff
Response time under growing context outruns what downstream systems tolerate. Nine seconds a step is non-functional, whatever the output says.
Repetition & persistent failure
Recycles prior output. One bad step is noise; three in a row is a state the system has entered.

The intervention ladder

Each failure mode that trips moves the agent one rung up the ladder. Abort is reached when failure persists — three consecutive failed steps under the same conditions — and the harness stops the agent before the output is used.

The finding

Benchmark scores and operational reliability diverge. A model that looks safe in a one-shot probe can still fail a long session.

Across seven language models — five run locally, two via API — and six runs each, every model landed in the same intervention regime on every run; the classification is reproducible. Then the benchmark and the stress test disagreed: Mistral-7B logged fewer benchmark contract failures than its counterparts, yet aborted under rolling stress, while models with worse benchmark scores stayed stable. A procurement process relying on benchmark scores alone would have made the wrong call.

The audit trail

When LTE intervenes it emits a structured record — which signals tripped, when, and what intervention state the agent reached — and retains it.

LTE assigns no blame. It records what an organisation knew about a model before it went live — which signals tripped, and when. Liability turns on foreseeability: whether the problem could have been caught in advance. Catching it needs an instrument that runs before deployment — and most deployments don't have one.
