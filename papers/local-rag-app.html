Local RAG App

Drag in your documents and draft or edit sections with a Mistral 7B model that runs entirely on your own machine.

What it is

A private RAG stack on your own machine

The app drafts and edits documents from your source material and never sends a byte to the cloud. Four properties make it useful where hosted tools aren't.

🔒 It stays local

The model and your documents both run on your device. The only network call is the one-time model download; after that you can pull the plug and it still works.

⚡ It runs on a laptop

Mistral 7B is quantized to 4 bits and runs on the CPU, adapting to however many cores you have. No GPU required.

📚 It cites your sources

Every draft is built from chunks retrieved out of your own files, and the excerpts it used are listed alongside the output. You can see exactly what informed each answer.

🛠️ It's yours to change

The pipeline is a few hundred lines of readable Python. Swap in a different model, rewrite the prompts, or extend the retrieval — nothing is hidden behind a service.

Get it

Download and run it

Grab the repository, then follow the three steps below. You don't need to read the code to use it — the launcher scripts do the work.

Local-RAG-App on GitHub

Clone it or download the ZIP, unzip, and run the launcher for your OS.

⬇ Download from GitHub Get the ZIP ↗

The first launch downloads a ~4.3 GB model, so it takes a while. That's expected, and it only happens once.

  1. Install Docker Desktop

    If you don't have it yet, download Docker Desktop ↗, install it, then open it and wait until it reads "Docker is running."

  2. Launch the app

    Run the launcher for your OS. There's no command line to touch; the script builds the container and opens the app for you.

    🪟 Windows

    Double-click run.bat. A terminal window opens — leave it open.

    🍎 macOS

    Right-click run.command → Open, then Open again when prompted.

    macOS blocking the file? Open Terminal and run: 
    xattr -dr com.apple.quarantine run.command

    Prefer the command line? From the repo folder:

    docker compose up        # or: streamlit run app.py

  3. Wait for first-run setup, then open the app

    On first launch the terminal builds the container, downloads the model (~4.3 GB), then opens your browser. This can take 5–60 minutes depending on machine and connection.

    When ready, the app lives at http://localhost:8501 ↗.

    Every run after the first is fast and fully offline — the model is already cached.
Using the app

Upload, then draft or refine

The workflow is three moves: load your sources, name the section you're writing, and let the app draft or continue it from your material.

1 · Upload your documents

Drop in one or more files — .md, .txt, .pdf, or .docx. They're chunked and embedded automatically (you'll see an "Embedding documents…" spinner). At least one document is required before anything else unlocks.

2 · Set a section title & optional prompt

Enter a Section title (e.g. Introduction) — this is the topic the app retrieves relevant excerpts for. Add optional instructions to steer tone, length, or focus.

3 · Draft or edit — two tabs

📝 Draft section

Click 🚀 Draft this section. The model streams a draft grounded in your top matching excerpts. Tweak it inline, then download it as .md. The source excerpts used are listed below the draft.

🔧 Edit / Continue

Paste an existing draft, then either 💡 Get edit suggestions (clarity & coherence) or 👉 Suggest continuation (what to write next) — both grounded in your sources.

Every output cites the source excerpts it drew from, so you can see exactly which parts of your documents informed the text.
Shutting down

Stop cleanly when you're done

  1. Click 💣 Shutdown in the sidebar

    This stops the app and clears all loaded files — nothing you uploaded persists.

  2. Close the terminal window

    Once shutdown completes, close the terminal manually. On Windows, also close out of Docker Desktop if you're finished.

How it works · light architecture

A classic RAG loop, all local

One Docker container, five stages, no network after setup.

Uploadmd · txt · pdf · docx
→
Chunk~300-word chunks
→
Embedall-MiniLM-L6-v2
→
RetrieveFAISS top-n
→
GenerateMistral 7B

When you draft or edit, the app builds a query from your section title and prompt, finds the most relevant chunks of your documents via a FAISS similarity search, packs those excerpts into the prompt as context, and streams the answer from a local Mistral 7B model via llama.cpp. Because the excerpts come from your files, the output stays grounded in your material.

Under the hood

ComponentWhat it does
LLMMistral-7B-Instruct v0.1, 4-bit (Q4_K_M) GGUF, run on CPU via llama-cpp-python with a 2048-token context.
Embeddingssentence-transformers/all-MiniLM-L6-v2 turns each chunk into a vector.
RetrievalFAISS IndexFlatIP (inner-product) returns the top-n most similar chunks.
ChunkingSentence-aware splitting into ~300-word chunks with overlap; chunks under ~40 words are dropped.
UIStreamlit app with streaming responses and downloadable drafts.

Swapping the model

Want something smaller or larger? Point the app at a different GGUF file. Update GGUF_MODEL_PATH in docker-compose.yml (or your environment) and place the new .gguf in models/. Lighter options like TinyLLaMA or Phi-2 run faster on modest hardware.

Troubleshooting

Common snags

SymptomFix
Stuck on the onboarding / "model not found" screenConfirm models/mistral-7b-instruct-v0.1.Q4_K_M.gguf exists and is ≥ 4 GB. Re-run bash scripts/download_model.sh if needed.
First launch is taking foreverExpected — the container build plus a ~4.3 GB download can take 5–60 min. Keep the terminal open and watch its progress.
macOS won't open run.commandRun xattr -dr com.apple.quarantine run.command in Terminal, then try again.
Low disk spaceMove the model to an external drive and symlink models/ to it.
Responses feel slowLower Max tokens and top_n, or swap in a smaller model. Speed scales with your CPU.
Browser didn't openManually visit localhost:8501 ↗ once the terminal says it's ready.
